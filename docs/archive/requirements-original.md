# 校园问答助手需求文档（PRD）

> 版本：v1.0
> 日期：2026-04-08
> 项目名称：校园问答助手开发项目

---

## 1. 项目概述

### 1.1 项目背景
校园问答助手是一个基于大语言模型（LLM）和检索增强生成（RAG）技术的智能问答系统，旨在为校园师生提供准确、高效的校园知识问答服务。系统通过构建校园知识库，结合AI大模型能力，实现智能化的问答交互。

### 1.2 项目目标
- 构建一个完整的校园知识问答系统
- 支持用户注册、登录、权限管理
- 支持知识库文档的上传、管理和检索
- 基于RAG技术实现智能问答
- 提供友好的Web交互界面

### 1.3 系统架构
系统采用前后端分离架构：
- **前端**：用户交互界面
- **后端**：业务逻辑处理
- **数据库**：用户数据、文档元数据、问答记录存储
- **向量库**：文档向量存储与相似度检索（FAISS等）
- **LLM**：大语言模型接口调用

## 2. 功能需求

### 2.1 用户管理模块

#### 2.1.1 用户注册
- 支持新用户注册账号
- 注册信息包括：用户名、密码、邮箱等基本信息
- 用户名唯一性校验
- 密码安全存储（加密）

#### 2.1.2 用户登录/退出
- 支持用户账号密码登录
- 登录成功后生成Token用于身份认证
- 支持用户安全退出，Token失效处理

#### 2.1.3 用户查询
- 支持查询用户列表（管理员权限）
- 支持按用户名等条件筛选查询
- 分页展示用户数据

#### 2.1.4 用户信息修改
- 支持用户修改个人基本信息
- 支持修改密码
- 支持管理员修改其他用户信息

#### 2.1.5 用户删除/禁用
- 支持管理员删除用户账号
- 支持禁用/启用用户账号
- 删除用户时关联数据处理

#### 2.1.6 权限控制
- 登录状态校验
- 访问权限控制（区分普通用户和管理员）
- 未登录用户访问限制

### 2.2 知识库管理模块

#### 2.2.1 文档上传
- 支持上传校园相关文档（PDF、Word、TXT等格式）
- 文档元数据记录（文件名、上传时间、大小等）
- 上传进度显示

#### 2.2.2 文档列表
- 展示已上传文档列表
- 支持按名称、时间等条件筛选
- 分页展示

#### 2.2.3 文档更新/重新处理
- 支持替换已有文档
- 重新触发文档处理流程（切分、向量化）

#### 2.2.4 文档删除
- 支持删除文档
- 删除文档时同步清理向量库中对应数据

#### 2.2.5 文档处理（后台）
- **文本切分（Chunk）**：将文档内容切分为适当大小的文本块
- **向量化处理（Embedding）**：将文本块转换为向量表示
- **向量库构建**：将向量存储到FAISS等向量数据库中，建立索引

### 2.3 AI问答模块（RAG核心）

#### 2.3.1 RAG核心流程
- **检索阶段**：根据用户问题，在向量库中检索TopK相关文档片段
- **生成阶段**：将检索结果作为上下文，结合Prompt调用LLM生成回答

#### 2.3.2 检索模块
- 用户问题向量化
- 向量相似度匹配（TopK）
- 支持调整检索参数（如K值）

#### 2.3.3 Prompt设计与上下文拼接
- 设计优化的Prompt模板
- 将检索到的文档片段与用户问题拼接为完整上下文
- 支持Prompt参数配置

#### 2.3.4 大模型API调用
- 支持调用主流大语言模型API（如OpenAI、文心一言等）
- 支持模型参数配置（temperature、max_tokens等）
- API调用异常处理与重试机制

#### 2.3.5 问答接口（/chat）
- 提供统一的问答接口
- 接收用户问题，返回AI回答
- 支持流式输出（可选）

#### 2.3.6 问答记录管理
- 保存问答历史记录（问题、回答、时间、用户等）
- 支持查询历史问答记录
- 支持按用户、时间等条件筛选

#### 2.3.7 问答效果测试与优化
- 支持问答效果测试
- 收集反馈用于优化

### 2.4 前端页面模块

#### 2.4.1 登录页
- 用户名、密码输入
- 登录按钮
- 跳转注册页链接

#### 2.4.2 注册页
- 注册信息填写表单
- 注册按钮
- 跳转登录页链接

#### 2.4.3 用户管理页面（管理员）
- 用户列表展示
- 用户增删改查操作
- 权限管理

#### 2.4.4 知识库管理页面
- 文档上传功能
- 文档列表展示
- 文档更新、删除操作
- 文档处理状态显示

#### 2.4.5 聊天页面
- 问答交互界面
- 消息展示（用户问题、AI回答）
- 输入框发送消息
- 历史记录展示

## 3. 非功能需求

### 3.1 性能需求
- 问答响应时间：普通问题3秒内返回
- 支持并发用户访问
- 向量检索效率满足实时性要求

### 3.2 安全需求
- 用户密码加密存储
- Token机制保障接口安全
- 防止SQL注入、XSS等攻击
- 文件上传安全检查

### 3.3 可用性需求
- 系统7×24小时可用
- 关键操作有错误提示
- 友好的用户界面

### 3.4 可扩展性需求
- 支持后续接入更多LLM模型
- 支持知识库文档类型扩展
- 支持功能模块扩展

## 4. 数据需求

### 4.1 数据库设计

#### 用户表（users）
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键，自增 |
| username | VARCHAR | 用户名，唯一 |
| password | VARCHAR | 加密密码 |
| email | VARCHAR | 邮箱 |
| role | ENUM | 角色（admin/user） |
| status | ENUM | 状态（enabled/disabled） |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

#### 文档表（documents）
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键，自增 |
| filename | VARCHAR | 文件名 |
| file_path | VARCHAR | 文件存储路径 |
| file_size | INT | 文件大小 |
| status | ENUM | 处理状态 |
| created_at | TIMESTAMP | 上传时间 |
| updated_at | TIMESTAMP | 更新时间 |

#### 问答记录表（chat_records）
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键，自增 |
| user_id | INT | 用户ID |
| question | TEXT | 用户问题 |
| answer | TEXT | AI回答 |
| created_at | TIMESTAMP | 问答时间 |

### 4.2 向量库
- 使用FAISS等向量数据库存储文档向量
- 支持向量相似度检索
- 向量维度与Embedding模型匹配

## 5. 接口需求

### 5.1 用户接口
- `POST /api/users/register` - 用户注册
- `POST /api/users/login` - 用户登录
- `POST /api/users/logout` - 用户退出
- `GET /api/users` - 用户列表查询
- `GET /api/users/{id}` - 用户详情查询
- `PUT /api/users/{id}` - 用户信息修改
- `DELETE /api/users/{id}` - 用户删除

### 5.2 知识库接口
- `POST /api/documents` - 文档上传
- `GET /api/documents` - 文档列表查询
- `PUT /api/documents/{id}` - 文档更新
- `DELETE /api/documents/{id}` - 文档删除

### 5.3 问答接口
- `POST /api/chat` - 问答请求
- `GET /api/chat/history` - 问答历史查询
