# 校园问答助手项目总结

## 项目背景

校园制度、办事流程和常见问题分散在不同文档中，查找成本较高。本项目构建了一个面向校园场景的智能问答系统，让用户通过自然语言获得基于校内知识资料的回答，并能查看答案引用依据。

## 最终成果

系统形成了完整的前后端应用：

- 用户注册、登录与角色权限控制。
- 管理员上传和维护知识库文档。
- 文档解析、切分、向量检索和 RAG 回答。
- 同步与 SSE 流式问答，多轮上下文与停止生成。
- 引用来源展示，便于核验答案依据。
- 会话历史保存、分页、搜索、重命名和删除。
- 科幻风响应式界面，兼顾桌面与移动端。
- 自动测试、冒烟测试及 Docker/Nginx 部署配置。

## 技术方案

前端采用 React、TypeScript 和 Vite；后端采用 FastAPI、SQLAlchemy 与 SQLite；AI 模块通过向量检索选取相关知识片段，再将片段和对话上下文交给大语言模型生成答案。部署层使用 Docker Compose，Nginx 同时负责静态页面发布、API 代理和流式传输。

## 项目亮点

1. 回答可追溯：每次回答携带文件、页码与原文片段。
2. 体验完整：既有流式反馈，也可主动停止生成和管理历史会话。
3. 权限清晰：普通用户和管理员功能边界明确。
4. 交付可复现：提供容器化配置、测试报告和自动冒烟脚本。
5. 异常可感知：模型不可用、权限不足或参数错误时给出明确反馈。

## 测试结论

后端自动测试 4/4 通过，语句覆盖率 77%；真实模型冒烟测试和 API 性能基线通过；前端生产构建与代码检查通过；容器编排配置验证通过。核心业务链路满足任务验收要求。

## 不足与后续方向

- 增加更多格式的文档解析和 OCR 能力。
- 使用专用向量数据库提升大规模知识库性能。
- 建立答案反馈、热点问题和质量评估体系。
- 增加模型降级策略、限流、审计日志和数据备份。
- 在真实校园数据上持续评估召回率与答案准确率。

## 总结

项目从需求、设计、开发、联调到测试和部署形成了完整闭环。最终版本不仅能完成校园知识问答，也具备引用核验、历史管理和标准化部署能力，可作为后续扩展校园智能服务的基础。
