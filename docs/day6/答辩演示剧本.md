# 校园问答助手答辩演示剧本

建议时长：6—8 分钟

## 1. 项目介绍（40 秒）

“校园资料通常分散在不同文档中，我们开发了校园问答助手。用户可以直接提问，系统会从校内知识库检索依据并生成答案，同时展示引用来源，避免答案无法核验。”

## 2. 首页与登录（40 秒）

展示科幻风首页和移动端适配，说明普通用户与管理员权限不同。使用演示账号登录。

## 3. 知识库管理（60 秒）

以管理员身份打开知识库页面，展示已有文档、处理状态和上传入口。说明上传后系统会自动解析、切分并建立检索索引。

## 4. 核心问答（2 分钟）

1. 提问一个能够在演示资料中明确找到答案的问题。
2. 展示流式生成效果。
3. 展开引用来源，核对文件名、页码和原文片段。
4. 继续追问一个依赖上文的问题，展示多轮上下文。
5. 可补充演示“停止生成”按钮。

建议提前准备的问题：

- “请根据知识库说明某项校园业务的办理流程。”
- “刚才流程中需要准备哪些材料？”

问题应与现场导入的资料一致，不要使用资料中不存在的信息。

## 5. 历史记录（50 秒）

刷新或切换页面后重新打开刚才的会话，展示消息和引用仍然存在；演示搜索和重命名会话。

## 6. 技术与测试（60 秒）

简要说明 React 前端、FastAPI 后端、RAG 检索增强生成和 Docker 部署。展示测试结论：后端 4/4 通过、覆盖率 77%、真实模型冒烟测试和性能基线通过、前端构建与检查通过。

## 7. 总结（30 秒）

“项目完成了从知识导入、检索问答、引用核验到历史管理的完整闭环，并提供自动测试和容器化部署。后续可以继续扩展 OCR、向量数据库和答案反馈评估。”

## 演示前检查

- 模型密钥、网络和账户额度正常。
- 导入至少一份适合提问的校园资料。
- 提前运行冒烟测试。
- 浏览器缩放为 100%，关闭无关窗口和通知。
- 准备录屏或截图作为网络异常时的备用方案。
